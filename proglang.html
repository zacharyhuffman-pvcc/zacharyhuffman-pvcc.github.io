Programming Languages
Term Definition
Machine Language a binary programming language that computers handle locally on the processor
Debugging running code line by line to find and resolve problems within code
Assembler A tool that translates assembly language into machine code.
Assembly Language A low-level programming language that uses symbolic instructions to interact with computer hardware.
Translator Software that converts code written in one programming language into another language, such as from high-level to machine code.
Compiler A translator that converts the entire source code of a program into machine code before execution.
Interpreter A translator that executes code line by line, translating high-level instructions into machine code during runtime.
Cross-Platform Software Software that can run on multiple operating systems
Natural Language Languages spoken and written by humans English is an example
Formal Language a precise and well-defined set of rules and syntax used to write code, allowes computers to interpret and execute programming instructions
Machine Independence a piece of code can be executed on different types of computers without needing significant modifications
High-Level Programming Language Languages like Python or Java that are easy for humans to read and write, and abstract from hardware specifics.
Object-Oriented Programming (OOP) A paradigm based on "objects," which can contain data and code to modify the data.
Scripting Language Programming languages like Python or JavaScript that are often interpreted and used for automating tasks.
Data Types three main types:
  • Integer: Whole numbers
  • Float: Numbers with decimal points
  • String: text based data
Primitive Data Type Basic data types provided by python (can be for any language as well), such as int, float, str, and bool
Data Structure A way of organizing and storing data
Python Collection Data Structures Types:
  • List:a collection of data that is mutable, meaning we can add, remove and change the contents
  • Tuplea collection of data that is unmutable, meaning it is fixed and cannot be modified after creation
  • Set: an unordered collection of data thats stored as a single variable, can be changed but cannot store duplicates of the same data
  • Dictionary: an ordered collection of data, can be changed and cannot store duplicates
Function A block of code designed to perform a specific task, can be called and used again throughout a program.
Local Variable A variable assigned within a function, exists only within that function.
Global Variable A variable outside of functions, can be called and used anywhere within a program
Source Program The original code written for a website or program, is the very DNA of its functionality
Object Program The output of a compiler or assembler, in machine code.
