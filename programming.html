Programming Algorithms & Software Testing
Topic Description
Phases of Computer Program Development
  • Phase 1: Understand the problem
  • Phase 2: Design an algorithm to solve the problem
  • Phase 3: Write the program code in appropriate language based on the algorithm
  • Phase 4: Enter the program and save
  • Phase 5: Execute the program
  • Phase 6: Test the program, find errors and debug, then retest
  • Phase 7: Document and evaluate the program
Programming Structures Sequential Structure: Top-down execution
Decision Structures: if/then/else
Iterative (Looping):
  • while loop: continue looping as long as something is true
  • repeat/until loop: continue looping until something is true
  • for loop: loop a given number of times
Recursive Structures: Repeat a set of instructions as a subtask of itself
Repetition continues until: degenerative case, which is a termination condition, is met
(degenerative case is also called a "base case")
Two Classic Programming/Database Problems Sorting: Descending/Ascending order
  • Sorting numbers
  • Sorting strings (in accordance with ASCII values)


Searching: Find specific data that meet certain criteria

In Database:

  • Queries:search criteria are saved so they don't have to be recreated

  • Sequential Search: Searches from top to bottom or bottom to top, one item at a time

Two Classic Program Goals

Efficiency: Program takes minimal time and/or resources
Correctness: Program runs as expected and works under all expected conditions

Kinds of Tests
  • Acceptance testing: Verifying whether the whole system works as intended.
  • Integration testing: Ensuring that software components or functions operate together.
  • Unit testing: Validating that each software unit runs as expected. A unit is the smallest testable component of an application.
  • Functional testing: Checking functions by emulating business scenarios, based on functional requirements. Black-box testing is a common way to verify functions.
  • Performance testing: Testing how the software runs under different workloads. Load testing, for example, is used to evaluate performance under real-life load conditions.
  • Regression testing: Checking whether new features break or degrade functionality. Sanity testing can be used to verify menus, functions and commands at the surface level, when there is no time for a full regression test.
  • Stress testing: Testing how much strain the system can take before it fails. Stress testing is considered to be a type of non-functional testing.
  • Usability testing:Validating how well a customer can use a system or web application to complete a task.
Purpose of Software Testing

Software testing helps to detect architectural flaws, poor design decisions, incorrect functionality, security vulnerabilities, and scalability issues.

Testing Best Practices
  • Continuous testing: Ongoing testing throughout development, using automated or manual methods to improve design and reduce risks.
  • Configuration management: Tracks project assets and code, ensuring compliance and accessibility for team members to hopefully reduce managerial overhead.
  • Service virtualization:a simulation of missing or not available yet code, so that software can go into a testing environment as to test different configurations and scenarios
  • Defect tracking or "bug tracking": Tracks defects during software development to improve quality.
  • Metrics and reporting: Monitors and shares project health and status during testing cyle.
