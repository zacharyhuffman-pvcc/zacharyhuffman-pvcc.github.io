Software Engineering Terms
Term Definition
Software Engineering part of computer science that focuses on the design, development and testing of software applications
Integrated Development Environment (IDE) an application where software developers can edit, develop, and debug code
Association for Computing Machinery (ACM) the largest educational and scientific group of tech individuals
Purpose of ACM to advance the world of computing technology through resource sharing, events, and dialoge
Benefits of ACM Membership Get access to their library of data, training material, personal networking in the field and ability to speak at events and publish papers
Student ACM Membership Offers students learning resources, scholarships, and mentoring opportunities.
Institute of Electrical and Electronics Engineers (IEEE) A global organization helping advance tech through research and collaboration different from ACM as IEEE focuses on more broad engineering topics
Purpose of IEEE To advance the development of technology for the betterment of humanity
Benefits of IEEE Membership Access to education literature, events, and a global network of peers.
Student IEEE Membership Gives students tools to learn, connect with professionals, and find career opportunities.
Software Life Cycle: Development Phase Steps like planning, building, and testing software during its creation.
Requirements Analysis Figuring out what users need from the software being developed.
Design Mapping out how the software will work and look.
Implementation Writing the actual code to make the software.
Testing Checking that the software works as intended and fixing bugs or issues.
Modular Implementation Breaking the program into smaller chunks to make development easier.
Unified Modeling Language (UML) standardized modeling language that offers visual diagrams that explain how a software system works.
Use Case A simple explanation of how a user should interact with the system.
Use Case Diagram A drawing showing how users and systems interact.
Data Relationships
  • One-to-One:A piece of data connects to one other piece of data
  • One-to-Many: A piece of data connects to several other pieces
  • Many-to-Many: Data is interconnected through a spiderweb of connections.
Design Pattern A common way to solve a software problem that works well.
Software Quality Assurance (SQA) Making sure the software is high-quality through testing and checks.
Software Testing
  • Glass Box Testing: Testing with full access to the code, focusing on its structure and logic.
  • Black Box Testing: Testing the software's functionality without knowing how the code works.
  • Beta Testing: Testing done by users in a controlled release to find bugs and gather feedback before the final release.
  • Alpha Testing: Testing done by the development team to catch bugs before letting users try it.
Documentation
  • User Documentation: Instructions on how to properly use software.
  • System Documentation: Details for developers on how to fix or update the software.
  • Technical Documentation: Notes showing more technical aspects of software, essentially the blueprints
Intellectual Property Law Laws in place to protect the ideas and creative works of an individual from being stolen or reproduced.
Software License A legal contract that outlines how people can use & share software
Software Disclaimer developers formal notice before a user can use software that if any harm or damage results from the use of the developers software, the user or users accept responsibility
Best Defenses Against Software Liability Claims Having clear licenses, testing thoroughly, and documenting everything well.
